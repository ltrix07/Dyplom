# Система статистичного моніторингу розвитку діджитал-процесів в Україні

Цей репозиторій містить код, який я використовую в дипломній роботі для побудови системи статистичного моніторингу розвитку діджитал-процесів в Україні на основі відкритих офіційних даних.

Проєкт автоматично:

- зчитує «сирі» статистичні дані з кількох джерел;
- приводить їх до єдиного табличного формату;
- формує панель показників за роками;
- розраховує інтегральний індекс розвитку діджитал-процесів відносно базового року;
- створює підсумкові таблиці та графіки для подальшого аналізу.

## Що саме вимірює система

У поточній версії аналізуються чотири показники, які відображають розвиток діджитал-процесів в Україні:

- **`internet_users_share`** – частка осіб, які користуються Інтернетом (% населення);
- **`internet_households_share`** – частка домогосподарств з доступом до Інтернету вдома;
- **`firm_website_share`** – частка підприємств, що мають власний вебсайт;
- **`ecommerce_firms_share`** – частка підприємств, які здійснюють електронну торгівлю.

Ці показники комбінуються в інтегральний індекс, який показує, як змінюється рівень розвитку діджитал-процесів у часі відносно базового року (за замовчуванням — **2019**).

## Структура проєкту

Організація файлів може трохи відрізнятися, але загальна логіка така:

```text
.
├─ app/
│  ├─ config.py           # налаштування: шляхи, список показників, базовий рік тощо
│  ├─ prepare_raw.py      # підготовка «чистих» файлів із сирих датасетів
│  ├─ merge_sources.py    # об'єднання всіх джерел у спільну панель
│  ├─ preprocessing.py    # попередня обробка, фільтрація років з браком даних
│  ├─ analytics.py        # розрахунок темпів зростання
│  ├─ index_builder.py    # обчислення індексів відносно базового року та інтегрального індексу
│  ├─ reporter.py         # збереження результатів, побудова графіків
│  └─ data_loader.py      # допоміжні функції для завантаження таблиць
│
├─ data/
│  ├─ raw/                # сирі дані та проміжні «чисті» CSV
│  │  ├─ households_with_internet_itu.csv
│  │  ├─ share-of-individuals-using-the-internet.csv
│  │  ├─ dataset_...ICT_ENTRP_...csv        # файли Держстату
│  │  ├─ households_internet_ua.csv         # згенеровано prepare_raw.py
│  │  ├─ internet_users_ua.csv              # згенеровано prepare_raw.py
│  │  ├─ ict_firms_website.csv              # згенеровано prepare_raw.py
│  │  └─ ict_firms_ecommerce.csv            # згенеровано prepare_raw.py
│  └─ digital_indicators_ua.xlsx            # інтегрована панель показників
│
├─ output/
│  ├─ results.xlsx         # підсумкова таблиця з індексами
│  ├─ indicators.png       # графік динаміки окремих показників
│  └─ digital_index.png    # графік інтегрального індексу
│
└─ main.py                 # головний сценарій запуску
```

## Технології

- **Мова:** Python 3.10+
- **Бібліотеки:**
  - `pandas` — робота з табличними даними;
  - `matplotlib` — побудова графіків;
  - `openpyxl` — збереження даних у форматі Excel.

## Підготовка даних

### Крок 1. Покласти «сирі» файли в `data/raw/`

У `data/raw/` мають лежати файли з офіційних джерел, наприклад:

- CSV / Excel із показником **Households with Internet access at home (%)** (ITU);
- CSV із показником **Individuals using the Internet (% of population)** (Our World In Data / World Bank / ITU);
- один або кілька файлів Держстату щодо:
  - частки підприємств, що мають вебсайт;
  - частки підприємств, які здійснюють електронну торгівлю.

Назви файлів можна змінювати — головне не забути узгодити їх у `app/prepare_raw.py` та `app/config.py`.

### Крок 2. Згенерувати «чисті» CSV для України

```bash
python app/prepare_raw.py
```

Після цього в `data/raw/` мають з’явитися файли:

- `households_internet_ua.csv`
- `internet_users_ua.csv`
- `ict_firms_website.csv`
- `ict_firms_ecommerce.csv`

Це вже готові ряди лише по Україні у форматі `year, value`.

## Основний сценарій

Після підготовки «чистих» даних можна запускати головний сценарій:

```bash
python main.py
```

Що при цьому відбувається:

1. Будується або завантажується інтегрована панель показників (`data/digital_indicators_ua.xlsx`).
2. Відбираються роки, для яких є повний набір показників (на момент написання роботи — 2018, 2019, 2021).
3. Додаються ланцюгові темпи зростання.
4. Розраховуються індекси відносно базового року (за замовчуванням 2019) та інтегральний індекс.
5. Результати зберігаються у `output/results.xlsx`.
6. Створюються два графіки:
   - `output/indicators.png` — динаміка окремих показників (максимально доступний період);
   - `output/digital_index.png` — інтегральний індекс за роками з повними даними.

## Налаштування

Основні параметри змінюються у файлі `app/config.py`:

- шляхи до папок `DATA_DIR`, `RAW_DIR`, `OUTPUT_DIR`;
- файл із панеллю `PANEL_FILE` та назва аркуша `PANEL_SHEET_NAME`;
- колонка з роком (`YEAR_COLUMN`);
- список показників для індексу (`INDEX_INDICATORS`);
- базовий рік для розрахунку індексів (`BASE_YEAR_FOR_INDEX`);
- перелік джерел даних (`RAW_SOURCES`), де задаються конкретні шляхи до CSV/Excel та відповідні назви колонок.

## Обмеження

- Повний набір значень для всіх чотирьох показників одночасно є лише для обмеженої кількості років. На момент написання дипломної роботи інтегральний індекс коректно розраховується лише для **2018, 2019 та 2021** років.
- Система не займається «вигадуванням» даних і не заповнює пропуски штучно — такі роки просто виключаються з розрахунку індексу.
- Набір показників підібрано під конкретні цілі дипломного дослідження. За потреби його можна розширювати (наприклад, додати показники щодо використання е-урядових послуг тощо).
